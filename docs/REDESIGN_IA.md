# Sowing IA 재설계 — 동사 중심 1급 모드

> **결정 일자**: 2026-05-11
> **상태**: 제안 (Phase 13 작업 분해는 [ROADMAP.md Phase 13](../ROADMAP.md#phase-13) 참조)
> **계기**: `docs/gb-docs.md` (김교수 "지독한 기록" 영상 transcript) 와의 비교 분석 — 동사 중심 IA + 즉각적 거울 + 계획 mode 가 진입장벽 문제 해결책
> **ADR 영향**: ADR-014 (신규) — 명사 mode 는 저장 단위, 동사 mode 는 사용자 의도 단위. 둘 다 유지, 사용자에겐 동사만 보임.

---

## 1. 문제 진단 — 현재 IA 의 3가지 약점

현재 nav 10개 (`홈 · 메모 · 필기 · 기록 · 태그 · 검색 · 템플릿 · 🌱합성기 · 🕸그래프 · 설정`) 의 문제:

| 문제 | 증상 |
|---|---|
| **평면 분산** | 10개가 한 줄 — 시각적 hierarchy 0 |
| **명사·저장위치 분류** | "메모/필기/기록" 은 폴더 이름. 사용자는 "행위" 로 사고 |
| **신규 진입자 혼란** | "메모? 필기? 기록? 차이가 뭐지? 어디부터?" |

→ Sowing 의 강점 (마크다운 SoT, 30년 누적, 16 합성기) 이 모두 같은 평면에 있어 **무엇이 먼저인지** 가 안 보임.

비교: 김교수 "지독한 기록" 은 5개 동사 메뉴 (`대시보드 · 기록하기 · 기록 보기 · 피드백 · 계획`) — 사용자 머릿속 동사와 1:1. 베타 인터뷰 시 가장 자주 들을 피드백 ("왜 적어야 하는지 동기가 안 와요") 의 일부는 IA 가 직접 원인.

---

## 2. 새 IA — 5개 동사 (+ 홈)

| 1급 | 동사 | 사용자 머릿속 질문 |
|---|---|---|
| 🏠 **홈** | (대시보드) | "오늘 뭐 일어났지?" |
| 🖊 **글쓰기** | 적는다 | "지금 떠오르는 걸 어떻게 빨리 적지?" |
| 📚 **쓴 글 보기** | 본다·찾는다 | "그때 그거 어디 적었더라?" |
| 🗓 **쓸 글 계획** | 계획한다 | "내일·이번 주·이번 달 뭐 할까?" |
| 🪞 **자기 거울** | 회고한다 | "내가 어떤 사람이지? 패턴이 뭐지?" |
| ⚙ **설정** | 조율한다 | "이 앱 어떻게 더 잘 쓰지?" |

**핵심 변화**: 명사 (메모·필기·기록) → 동사 (쓰기·보기·계획·회고). 사용자 멘탈 모델과 1:1.

## 3. 각 1급 안의 2급 — 직관적 통합

### 🖊 글쓰기 — 입력 진입점 (결정 0)

```
🖊 글쓰기
├── ⚡ 빠른 메모              (Cmd+Shift+M 동일, 휘발성, 분류 0 결정)
├── 📖 책 기록                (제목·페이지·인용 슬롯)
├── 🎤 강의·연수 기록         (강사·주제·핵심)
├── 💭 감정 기록              (18종 감정 chip)
├── 👤 학생 관찰              (학생 chip + 본문)
├── 📝 필기 작성              (CodeMirror 에디터, 카테고리 결정)
└── 🎙 음성 입력              (Whisper 기반, 신규)
```

**통합 효과**:
- 기존 1급 "메모" + "필기" + (신규) 분류별 폼 + (신규) 음성 = 한 곳
- 신규 사용자가 "글쓰기" 한 번만 클릭하면 7가지 진입점이 직관적으로 펼쳐짐
- 모두 동일한 마크다운 + frontmatter `subtype` 으로 저장 (DB 호환, 옵시디언 호환)

### 📚 쓴 글 보기 — 회상·검색 (다축 진입)

```
📚 쓴 글 보기
├── 🕐 최근                    (시간순, 메모/필기/기록 통합)
├── 📁 카테고리별              (수직 분류 — 30_Records/{YYYY}/{cat}/)
├── 📊 카테고리 × 연도          (30년 매트릭스 ⭐ 핵심 차별점)
├── 📅 Timeline                (일별 평면, 색·점)
├── 🏷 태그                    (수평 색인)
├── 🕸 위키링크 그래프          (force-directed)
├── 🔍 검색                    (FTS5 풀텍스트)
├── 📆 이날의 회고              (cross-year — 같은 월·일)
└── 🏝 고립 노트                (#5 orphan-detection — 잊혀진 발견)
```

**통합 효과**:
- 기존 1급 "메모/필기/기록/태그/검색/그래프" + 보조 routes (timeline·by-category·이날의회고) = 한 곳
- "**회상 도구는 모두 여기**" 라는 명확한 약속
- 검색·태그·그래프 — 각각 다른 축이지만 모두 "이미 쓴 것을 다시 보기" 라는 공통 의도

### 🗓 쓸 글 계획 — 미래 (신규 mode)

```
🗓 쓸 글 계획
├── ✅ 오늘 할 일              (daily)
├── 📋 이번 주 계획             (weekly)
├── 🎯 이번 달 로드맵           (monthly)
├── 🏗 프로젝트                 (장기 — 책 쓰기, 단원 개발 등)
└── 🎓 학기 계획                (분기/학기)
```

**신규 4번째 1급 mode** — 폴더 `40_Plans/` 도입.

**핵심 가치**: 기록 → **실행** 연결. 지금 Sowing 은 회상에는 강하지만 "내일 뭐 할까?" 에 답하지 못함. 계획 mode 가 그 단절을 메움.

**propagation 옵션** (LLM 사용 시):
- 월간 로드맵 → "1주차에 A 배치할까요?" 제안 (자동 X, 사용자 클릭)
- 일일 계획 옆에 "어제 회고" inline (📚 쓴 글 보기 와 자연 연결)

### 🪞 자기 거울 — 회고·합성 (16+1 종)

```
🪞 자기 거울
├── 🌅 오늘의 자기              (매일 5축 분석 — 결정적 또는 LLM)
├── 📅 이번 주 회고             (#4 weekly-review)
├── 🌱 합성기 17종              ← 16 기존 + 1 신규 self-mirror
│   ├── 학생 디제스트
│   ├── 학기 회고
│   ├── 학생 묘사 변화
│   ├── ... (16종)
│   └── 🆕 5축 자아 분석         (지성·감정·습관·관계·에너지)
├── 📥 검토 대기                (`.sowing/synth/` 통합 inbox)
└── 📊 사용 지표 (베타)         (수락률·type 별·주별)
```

**통합 효과**:
- 기존 1급 "합성기" 확장
- 매일 자동 거울 (옵션) — 지독한 기록의 즉각성 흡수
- 검토 대기 inbox — 16+1종 합성 결과를 한 곳에서 일괄 검토

### ⚙ 설정 — 조율 (명료화)

```
⚙ 설정
├── 👥 학급 명단               (미언급 학생 감지)
├── 🔑 LLM 설정                (.env 키, 모델 선택 — 기존 분산을 통합)
├── 🪞 거울 옵션                (매일 자동 생성 on/off, 시간대)
├── 🎓 튜토리얼                 (다시 보기)
├── 💾 백업·동기화              (vault 경로, iCloud/Dropbox 안내)
└── ℹ 앱 정보                  (버전, 라이선스)
```

## 6. 마이그레이션 — 기존 데이터 + URL 100% 보존

### 폴더 구조 — 그대로

```
00_Inbox/        ← "글쓰기 → 빠른 메모" 진입점
20_Notes/        ← "글쓰기 → 필기 작성"
30_Records/      ← "쓴 글 보기 → 카테고리"
40_Plans/        ← "쓸 글 계획" (신규)
.sowing/synth/   ← "자기 거울 → 검토 대기"
```

ADR-001 (마크다운 SoT) 영향 0. 옵시디언 호환 유지.

### URL 라우팅 — 호환

```
기존 (북마크 보호):       /memos, /notes, /records, /tags, /search, /synth, /graph
새 진입점 (추가):         /write, /view, /plan, /mirror
```

기존 URL 모두 작동. 새 URL 은 추가 진입점 역할 — 사용자 북마크·외부 링크 깨지지 않음.

### Nav HTML 변경 (구현 예시)

```erb
<!-- 변경 전: views/layouts/application.erb -->
<nav>

<!-- 변경 후 (드롭다운 with 2급) -->
<nav class="nav-v2">
  <a href="/">🏠 홈</a>
  <details class="nav-dropdown">
    <summary>🖊 글쓰기</summary>
```

`<details>` element + CSS `:hover` / `:focus-within` — JS 0 으로 dropdown 구현. CLAUDE.md 의 "빌드 도구·JS 의존성 최소화" 원칙 유지.

새 ADR 초안:

> **ADR-014: 동사 중심 IA (저장 단위와 의도 단위 분리)**
>
> **컨텍스트**: Phase 11~12 16 합성기 + Phase 1 메모/필기/기록 mode 가 같은 nav 평면에 노출돼 신규 사용자 혼란.

>
> **결과**:
> - ✅ 신규 사용자 진입장벽 ↓
> - ✅ 옵시디언 호환 (저장 단위 변경 0)
> - ⚠ Nav 변경 시 베타 사용자 재학습 필요 — 변경 후 첫 진입 시 안내 모달

| Nav 항목 평균 hover 횟수 (찾기까지) | 3.2회 | < 1.5회 | 베타 테스터 사용성 테스트 |
| 합성기 사용률 (월 1회 이상) | ~30% | > 60% | `compute_synth_metrics` 출력 |

## 11. 핵심 통찰

> **명사 mode 는 저장 단위, 동사 mode 는 사용자 의도.**
>
> Sowing 의 강점 (마크다운 SoT, 30년 누적, 16 합성기) 은 모두 **명사** (메모·필기·기록·카테고리·태그). 사용자 머릿속은 **동사** (적다·보다·계획하다·회고하다). 그 간극을 nav 가 메워야 합니다.

| | 기존 IA | 새 IA |
|---|---|---|
| 노출 단위 | **저장 단위** (명사) | **의도 단위** (동사) |
| 사용자 부담 | 매번 매핑 (`적고 싶은 건… 메모? 필기?`) | 동사 클릭 — 시스템이 저장 단위로 매핑 |
| 정체성 보존 | — | 마크다운 SoT, ADR-013 그대로 |

→ **Sowing 의 마크다운 SoT 정체성은 유지, UX 진입장벽만 낮춤.**
