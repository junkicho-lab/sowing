## 1. 왜 Sowing 인가 — 5가지 약속

Sowing 은 다음 **5가지를 거부** 합니다 (ADR-013):

| 거부 | 의미 |
|---|---|
| 챗봇 X | 대화창 없음. 도구일 뿐, 친구가 아님. |
| 자동 글쓰기 X | LLM 이 사용자 대신 쓰지 않음. 사용자 입력이 항상 진실. |
| 클라우드 강제 X | 모든 데이터는 로컬 마크다운 파일. 클라우드 옵션은 사용자 선택. |
| 의인화 X | "선생님 오늘 힘드셨죠?" 같은 가짜 공감 0. |
| 자율 mutation 0 | LLM·자동 분석은 검토 대기 폴더 (`.sowing/synth/`) 에만 작성. 사용자 클릭 없이는 정식 기록이 되지 않음. |

대신 다음을 **약속** 합니다:

1. **마크다운 SoT (Source of Truth)** — 모든 데이터는 `.md` 파일. 옵시디언·iA Writer·VS Code 어디서든 열림.
2. **30년 누적 설계** — 매 학년 새 폴더가 아니라, `30_Records/{YYYY}/{카테고리}/` 한 흐름으로 누적.
3. **로컬 우선** — SQLite 인덱스는 캐시. 마크다운만 있으면 재구축 가능.
4. **외부 의존성 0 (LLM 옵션 제외)** — 인터넷 없이도 모든 기능 정상.
5. **사용자 명시 클릭** — 어떤 자동 산출물도 검토→수락 전엔 정식 기록 아님.

## 2. Nav — 5개 동사 + 홈·설정

Phase 13 (W25-T01) 부터 nav 가 **사용자 머릿속 동사** 와 1:1 매칭되도록 재구성됐습니다.

```
🏠 홈   🖊 글쓰기 ▾   📚 쓴 글 보기 ▾   🗓 쓸 글 계획 ▾   🪞 자기 거울 ▾   ⚙ 설정
```

| 1급 | 동사 | 사용자 머릿속 질문 | 2급 진입점 |
|---|---|---|---|
| 🏠 홈 | (대시보드) | "오늘 뭐 일어났지?" | 통계 / 성장 / 오늘 할 일 / 오늘의 자기 |
| 🖊 글쓰기 | 적는다 | "지금 떠오른 걸 어떻게 빨리?" | 빠른 메모 / 책·강의·감정·학생 / 필기 / 음성 |
| 📚 쓴 글 보기 | 본다 | "그때 그거 어디 적었더라?" | 최근 통합 / 카테고리 / 매트릭스 / Timeline / 태그 / 그래프 / 검색 |
| 🗓 쓸 글 계획 | 계획한다 | "내일·이번 주 뭐 할까?" | 일간 / 주간 / 월간 / 프로젝트 / 학기 |
| 🪞 자기 거울 | 회고한다 | "내가 어떤 사람이지?" | 합성기 17종 / 사용 지표 / **오늘의 자기 (5축)** |
| ⚙ 설정 | 조율한다 | "이 앱 어떻게 더 잘?" | 호칭 / 학급 명단 / LLM / 거울 옵션 / 백업 |

**핵심 변화**: 명사 mode (메모·필기·기록·계획·합성) 는 **저장 단위** (그대로 유지), 동사 mode (글쓰기·보기·계획·회고) 는 **사용자 의도** (nav 노출). 자세한 배경: [DECISIONS.md ADR-014](DECISIONS.md#adr-014-동사-중심-ia--명사-저장-단위-와-동사-의도-두-계층-명시-분리).

> **기존 URL 호환** — `/memos`, `/notes`, `/records`, `/tags`, `/search`, `/synth`, `/graph` 모두 그대로 작동합니다. 북마크·외부 링크 영향 0.

## 3. 기록의 큰 흐름 — 메모 → 필기 → 기록 → 계획

Sowing 의 4 mode 는 **회상 (휘발성 → 영구성)** + **실행 (계획)** 의 흐름입니다.

| Mode | 폴더 | 성격 | 시간 가치 |
|---|---|---|---|
| 💭 **메모** | `00_Inbox/` | 떠오르는 한 줄, 임시 | 며칠 ~ 몇 주 |
| 📝 **필기** | `20_Notes/{카테고리}/` | 정리·연결, 작업 중 | 한 학기 |
| 📖 **기록** | `30_Records/{YYYY}/{카테고리}/` | 보존, 분류 | 30년 누적 |
| 🗓 **계획** *(W27 신규)* | `40_Plans/{period}/{date}.md` | 미래 청사진 (일·주·월·프로젝트·학기) | 단기~학기 |

```
00_Inbox/        ← 메모: "민준이가 처음 발표 자원했다"        ↑ 회상
   ↓ (정리·확장)
20_Notes/        ← 필기: "학생 관찰: 민준" — 위키링크·태그
   ↓ (완성·분류)
30_Records/      ← 기록: 30_Records/2026/학생기록/...

40_Plans/        ← 계획: 40_Plans/daily/2026-05-12.md       ↓ 실행
                   ("내일 협동학습 평가 루브릭 정리")
```

흐름의 핵심:
- **회상**: 억지로 모든 메모를 기록으로 끌어올릴 필요 없다. 대부분의 메모는 휘발돼도 좋다 — 중요한 것만 자연스럽게 위로.
- **실행**: 회상에서 발견한 패턴이 다음 계획으로. 계획 → 실행 → 회상 (메모/필기) → 다시 계획. 30년 누적.

### 필기 목록 — `/notes`

![필기 목록 — 카테고리별 정리, 위키링크·태그 확인](screenshots/03-notes-list.png)

필기는 메모보다 정돈된 단계 — 카테고리가 이미 정해져 있고 (`lessons` / `meetings` / `books` / 또는 자유), 위키링크로 다른 노트들과 묶입니다. **CodeMirror 6 기반 마크다운 에디터** 가 위키링크 자동완성·실시간 프리뷰를 지원합니다.

## 4. 첫 5분 — 메모 한 줄로 시작

설치 직후 진입하면 대시보드 (`/`) 가 보입니다:

![대시보드 — 통계 + 성장 단계 + 합성기 검토 위젯](screenshots/01-dashboard.png)

대시보드 구성:
- **오늘 / 이번 주 / 이번 달** — 작성 통계
- **연속 기록** — 🔥 streak (매일 하나씩 쌓는 동기부여)
- **성장 단계** — 🌱 새싹 → 🌿 줄기 → 🌳 거목 (30년 누적 시각화)
- **최근 메모** — 빠른 회상
- (데이터 누적 시) **이날의 회고** — 같은 월·일 과거 entries
- **합성기 검토** — 16종 합성기의 검토 대기 카운트

### 메모 작성 — 어디서든 ⌘+Shift+M

가장 빠른 시작 방법:

1. 어떤 화면에서든 **`⌘ + Shift + M`** (Mac) / **`Ctrl + Shift + M`** (Windows/Linux) 누르기
2. 모달 창에 한 줄 작성
3. **Enter** — 즉시 저장 (`00_Inbox/` 폴더에 마크다운 파일로)

이게 전부입니다. 분류 0, 제목 0, 카테고리 0 — **떠오르는 그대로**.

### 메모 목록 — `/memos`

![메모 목록 — 시간순, 태그 + 위키링크 inline 표시](screenshots/02-memos-list.png)

- 최신순 정렬 — 가장 최근이 위
- 본문에서 자동 추출된 **태그** (`#수업`, `#회고`) 와 **위키링크** (`[[학생 관찰: 민준]]`) 가 즉시 활성
- 아이콘: 📝 = 필기로 승격, 📖 = 기록으로 승격

> 💡 **승격(promote)**: 메모를 그대로 두지 않고 더 영구적인 mode 로 옮기는 동작. UI 의 📝 / 📖 아이콘 클릭으로 한 번에.

## 5. **범주(카테고리) 중심** — 분류의 정신

여기가 Sowing 의 **핵심 철학** 입니다.

다른 도구들 (옵시디언·Notion·Roam) 은 **자유 연결** 을 강조합니다. Sowing 은 다릅니다 — **범주가 먼저, 연결은 그 위에**.

### 왜 범주가 먼저인가

교사의 1년은 자연스러운 범주가 있습니다:

- **수업** — 매일 일어나는 본업
- **수업회고** — 그 날 끝난 후의 반성
- **학생기록** — 누군가에 대한 관찰
- **상담** — 학부모·학생과의 대화
- **평가** — 시험·과제 결과 분석
- **학급운영** — 일상 운영 (자리, 청소, 규칙)
- **학기회고** — 한 학기를 돌아본 큰 회고
- **연수** — 외부 학습
- **도덕수업** — 교과별 특화

**범주가 명확하면** 30년 후에도 찾을 수 있습니다. "2032년 학생 민준이 발표 변화" — 범주 (`학생기록`) + 시간 (`2032`) 두 축으로 즉시 도달.

### 기록 목록 — 범주 chip 으로 필터

![기록 목록 — 카테고리 chip 으로 즉시 필터](screenshots/04-records-list.png)

상단의 **카테고리 chip** (전체 / 상담 / 수업 / 수업회고 / 평가 / 학급운영 / 학기회고 / 학생기록) — 한 번 클릭으로 해당 범주만 필터.

각 카드:
- **카테고리 뱃지** (좌상단) — 범주 즉시 식별
- **날짜** (우상단)
- **제목 + 발췌**

### 카테고리 × 연도 매트릭스 — 30년 한 화면

기록 목록 우측 상단의 **📈 카테고리 × 연도** 클릭:

![카테고리 × 연도 매트릭스 — 한 화면에 30년 분포](screenshots/06-records-by-category.png)

- **행** = 카테고리 (빈도순)
- **열** = 연도
- **셀** = 해당 (카테고리, 연도) entry 수
- **셀 클릭** → timeline 에 해당 (카테고리, 연도) 만 필터

30년 후 이 매트릭스를 보면:
- 어떤 범주가 꾸준히 누적됐는지 (수업회고 = 매년 50건+)
- 어떤 해는 특정 범주가 비어있었는지 (2028년 상담 0 — 그 해 무슨 일이 있었나?)
- 어떤 범주가 새로 생겼는지 (2030년 "AI도구활용" 신설)

**이 매트릭스가 Sowing 의 가장 강력한 30년 회상 도구** 입니다.

### Timeline — 시간 평면 뷰

![Timeline — 연·월·일 평면, 31×N 그리드](screenshots/05-records-timeline.png)

- 일별로 entries 점/색 표시
- 카테고리·연도 필터 적용 (URL `?category=...&year=...`)
- 매트릭스에서 셀 클릭 시 자동 필터되어 진입

## 6. 태그 — 카테고리 횡단 색인

카테고리가 **수직 분류** (폴더 구조) 라면, 태그는 **수평 색인** 입니다.

### 태그 작성

본문에 `#태그명` — 어디서든.

```markdown
오늘 협동학습 도입 3일째. 모둠별 속도 차이가 큼. #회고 #협동학습
```

작성 시 자동으로 인덱싱돼 `/tags` 에 노출:

![태그 클라우드 — 16개 태그, 빈도 표시](screenshots/07-tags-cloud.png)

각 태그 chip 의 숫자 = 사용된 entries 수. 클릭하면 해당 태그를 가진 모든 entries (메모·필기·기록 무관) 가 시간순으로.

### 카테고리 vs 태그 — 언제 무엇을?

| 상황 | 카테고리 | 태그 |
|---|---|---|
| 본질적 분류 | ⭕ 폴더 자체가 카테고리 | ❌ |
| 횡단 주제 (여러 범주 공통) | ❌ | ⭕ `#협동학습` |
| 일회성 라벨 | ❌ | ⭕ `#준비` |
| 학생 이름 | ❌ (위키링크 사용) | △ `#학생관찰` |

규칙: **카테고리는 1개, 태그는 0~N개**. 한 기록의 본질은 하나지만, 가로질러 묶을 키워드는 여럿일 수 있습니다.

## 7. 위키링크 — 노트 간 연결

본문에 `[[다른 노트 제목]]` — 옵시디언 호환 구문.

```markdown
민준이가 오늘 처음으로 발표를 자원했다.
자세한 관찰은 [[학생 관찰: 민준]]에 정리할 것.
```

저장 즉시 자동으로:
- 링크가 파란색으로 활성 (UI 에서 클릭 가능)
- 대상이 없으면 회색 (미완성 링크 — 클릭 시 생성 프롬프트)
- **그래프** (`/graph`) 에 노드·엣지로 표시

### 그래프 시각화

![위키링크 그래프 — force-directed SVG, 카테고리 필터](screenshots/09-graph.png)

## 8. 30년 시나리오 — 시간 축의 회상

Sowing 의 가장 큰 차별점: **30년 후에도 동작합니다**.

### 4가지 cross-year 탐색 도구

1. **이날의 회고** (대시보드) — 오늘과 같은 월·일의 과거 연도 entries 자동 표시. "5월 11일 — 2024년에도 똑같은 갈등이 있었구나."
2. **카테고리 × 연도 매트릭스** (`/records/by-category`) — 위 §4 참조.
3. **Timeline** (`/records/timeline`) — 일별 평면, 색·점으로 누적 분포.
4. **합성기 전체기간 preset** — 합성 폼의 `📅 전체 기간 (30년)` 버튼 클릭 시 since/until 자동 채움.

### "30년 누적" 의 실제 의미

| 연차 | 시나리오 |
|---|---|
| 1년차 | 메모·필기 가벼움. 합성기 출력은 빈약. **이 때 사용 습관 형성** 이 가장 중요. |
| 3년차 | 카테고리별 100건+ 누적. "작년 이맘때 도덕수업 어떻게 했지" 검색 1초. |
| 5~10년차 | 매년 회고 가능. **합성기 self-patterns** (메타-합성) 가 본인의 집필 패턴 분석. |
| 20~30년차 | 매일 "이날의 회고" 가 자연스럽게 채워짐. 후임에게 인계할 자산. |

**핵심**: 1년차에 가장 가볍게 시작하세요. 매일 한 줄 메모로도 충분합니다.

## 9. 쓸 글 계획 mode — 4번째 1급 시민

Phase 13 W27-T01 신설. 회상 (메모·필기·기록) 이 과거 축이라면, **계획은 미래 축** 입니다.

```
🗓 쓸 글 계획 ▾
├── 📅 일간      — 오늘 할 일 (`40_Plans/daily/YYYY-MM-DD.md`)
├── 📋 주간      — 이번 주 계획 (`40_Plans/weekly/YYYY-Www.md`)
├── 🎯 월간      — 이번 달 로드맵 (`40_Plans/monthly/YYYY-MM.md`)
├── 🏗 프로젝트   — 장기 (책 쓰기·단원 개발 등, slug 명)
└── 🎓 학기      — 분기 단위 (`YYYY-S1` / `YYYY-S2`)
```

### 사용 흐름

1. `🗓 쓸 글 계획` 클릭 → `/plans?period=daily` (또는 4 period 중)
2. **＋ 새 계획** 버튼 → 제목 + 날짜 + 마크다운 본문 (todo list, 위키링크, 태그 모두 허용)
3. 저장 후 카드 표시 — 좌측 보라 border (Plan mode 의 색상 identity)
4. **✅ 완료** 버튼으로 done 토글 (사용자 명시 클릭만 — ADR-013)
5. 옵시디언에서 `40_Plans/...md` 직접 편집 가능 — `- [ ]` / `- [x]` todo 체크박스 native 지원

### 대시보드 "오늘 할 일" 위젯

오늘의 미완료 daily plan 이 있으면 대시보드 첫 화면에 자동 표시 — "오늘 무엇을 해야 하는지" 즉각 인지.

## 10. 자기 거울 — 매일 5축 자아 분석

Phase 13 W28 (17번째 합성기 + 위젯 + 자동 hook) 의 정수. "왜 매일 적어야 하는지" 에 대한 즉각적 동기부여.

### 5축

| # | 축 | 내용 |
|---|---|---|
| 1 | 🧠 **지성** | 자주 환기한 키워드 top 5 (조사 제거 + STOPWORDS 필터) |
| 2 | 💭 **감정** | POSITIVE/NEGATIVE 신호어 카운트 + 비율 (단정 X, 통계만) |
| 3 | 🔁 **습관** | 작성 시간대 분포 + 카테고리·모드 분포 |
| 4 | 🤝 **관계** | entity_mentions top 5 (학생·동료·학부모) |
| 5 | ⚡ **에너지** | 작성 빈도 + 일평균 + 작성한 날짜 수 |

### 기간 2종

- **daily**: 오늘 1일 — 즉각적 거울
- **weekly**: 이번 주 7일 — 한 주 패턴

### 매일 자동 생성 (opt-in)

`⚙ 설정 → 🪞 자기 거울` 에서 체크박스 켜면:
- 매일 대시보드 첫 진입 시 오늘 entries ≥ 3 이면 **자동 생성**
- 결과는 `.sowing/synth/self-mirror/daily-{date}.md` — 검토 대기 폴더
- 대시보드 위젯에 5축 요약 카드 즉시 표시 + `[5축 자세히 보기 →]`

### ADR-013 단정 거부

- "당신은 ~ 한 사람" 단정 X — 통계와 사실만
- "지쳤네요" 감정 단정 X — "부정 신호어 N건" 같은 *사실*
- LLM 모드도 5축 해석을 *후보* 로만 표현
- 자동 생성 결과도 검토 대기 폴더 — 정식 기록은 사용자 수락 클릭으로만

### vs SelfPatterns (#10)

| | SelfPatterns | SelfMirror (#17) |
|---|---|---|
| 기간 | 큰 기간 (학기·1년) | 짧은 기간 (1일·7일) |
| 깊이 | 4 섹션 (톤 변화·환기·burnout·다음 시도) | 5축 즉각적 거울 |
| 자동 생성 | X (수동) | ✅ daily_mirror_enabled |
| 대시보드 위젯 | X | ✅ |

### 17 type 분류

| 분류 | 합성기 | 용도 |
|---|---|---|
| 학생 중심 | 학생 디제스트, 학기 회고, **학생 묘사 변화** | 학생 개인 추적 |
| 수업·학습 | 수업 패턴, 수업 시리즈, **학습 진척 추이** | 교과·차시 분석 |
| 학부모·평가 | 상담 준비, **상담 패턴** (학기), 평가 누적, 연수 흡수 | 외부 기록 |
| 횡단 | 주간 회고, 고립 메모, 태그 클러스터, 계절성 | 자율 발견 |

Phase 13 W28-T01 의 17번째 합성기 **자기 거울** 은 §10 에서 상세 설명. 자세한 5축 결과 예:

![자기 거울 — 5축 결과 페이지](screenshots/23-synth-self-mirror.png)

### 결정적 vs LLM 모드

각 합성기는 두 모드를 지원:

| 모드 | 비용 | 속도 | 출력 |
|---|---|---|---|
| **결정적 (기본)** | 0 | 즉시 | 통계·인용 — 객관적 사실만 |
| **LLM (선택)** | $0.008~$0.12 | 5~30s | 자연어 패턴 분석 — 해석·역추론 |

LLM 모드는 `.env` 에 `ANTHROPIC_API_KEY=sk-ant-...` 설정 시 자동 활성. UI 에 체크박스 + 모델 드롭다운 (Haiku 4.5 / Sonnet 4.5 / Opus 4.7) 표시:

```
☑ 🌱 LLM 모드 — 자연어 패턴 분석
모델: [Haiku 4.5 — 저비용·빠름 (≈ $0.0080 / 합성 1건, 2~5s) ▼]
```

체크 안 하면 결정적 모드 (즉시·무료).

### 합성기 사용 흐름

1. `/synth` 진입
2. 16 type 중 하나 펼치기 (예: "학생 디제스트")
3. 폼 작성:
   - 학생 이름 (예: "민준")
   - 기간 (선택, 전체 기간 버튼)
   - (선택) LLM 모드 체크 + 모델 선택
4. **생성/재생성** 클릭
5. 결과가 `.sowing/synth/students/민준.md` 에 작성됨 (정식 기록 아님 — 검토 대기)
6. 카드에서 **자세히 보기** → 본문 확인 → **수락** or **거절**

### 합성 결과 카드

```
┌─ 학생 관찰: 민준 ─────────────────────────────────┐
│ [LLM 합성] [이번 주 새로 합성] Anthropic 2026-05-11 출처 7건 │
│                                                  │
│ # 학생 관찰: 민준                                 │

│                                                  │
```

3 액션:

## 14. 자주 묻는 질문

또는 옵시디언 / VS Code 로 `00_Inbox/{메모-id}.md` 직접 편집 — Sowing 이 파일 변경 감지 후 자동 인덱싱.

### 메모를 영구히 삭제하려면

UI 에서 메모 카드의 **🗑 거절** 버튼. `.sowing/trash` 로 이동 (30일 후 자동 삭제).

즉시 영구 삭제: `rm 00_Inbox/{메모-id}.md`.

### 위키링크 대상이 없어요 (회색)

`[[학생 관찰: 민준]]` 작성 시 대상이 없으면 회색 표시. 클릭하면 빈 노트 생성 프롬프트.

또는 일부러 placeholder 로 둘 수도 있습니다 — 나중에 노트를 만들면 자동 연결.

### 동시에 여러 기기에서 사용해도 되나요

가능하지만 **권장하지 않음**:
- vault 폴더를 iCloud/Dropbox 로 동기화 + 한 번에 한 기기에서만 사용 → 안전
- 두 기기 동시 사용 시 SQLite 동시 쓰기로 인덱스 충돌 가능

베타 기간엔 **데스크톱 1대** 권장.

### LLM 비용이 걱정돼요

| 사용 패턴 | 월 추정 비용 |
|---|---|
| 결정적만 사용 | $0 |
| 매주 LLM 합성 1회 (Haiku) | ≈ $0.03 |
| 매일 LLM 합성 1회 (Haiku) | ≈ $0.24 |
| 매주 Sonnet 합성 1회 | ≈ $0.10 |
| 매주 Opus 합성 1회 | ≈ $0.50 |

UI 에 매 합성마다 추정 비용 ($0.0080 / 합성 1건 형식) 표시 — 알고 누름.

### 베타 후 어떻게 되나요

- 데이터는 100% 본인 소유 (로컬 마크다운). Sowing 사용 중단해도 옵시디언 / VS Code 로 그대로 사용 가능.
- 베타 의견은 [BETA_GUIDE.md](BETA_GUIDE.md) 의 측정 기준에 반영됩니다.
- v1.0 정식 출시 시 베타 데이터는 그대로 호환됩니다.
