# Sowing 베타 테스터 모집 — 운영자용

> Phase 11/12 마일스톤 측정을 위한 베타 사용자 5명 모집 절차.

## 1. 모집 대상

**이상적 베타 사용자**:
- 한국 초·중등 교사 (현직)
- 평소 손글씨 또는 디지털로 학생 관찰·수업 회고를 1주에 3건 이상 작성
- 옵시디언·노션·로그시퀀스 등 마크다운/노트 도구 경험 있음 (필수는 아님)
- macOS / Windows / Linux 중 1대 보유, Ruby 4.x 설치 가능
- 한 학기 (3~6개월) 사용 + 인터뷰 1회 협조 가능

**우선순위**:
1. ROADMAP·ADR 작성에 참여한 사용자 (이미 컨텍스트 있음)
2. 옵시디언 한국 커뮤니티 (옵시디언 호환성을 즉시 평가 가능)
3. 일반 교사 (가장 광범위한 검증 — 도구에 익숙하지 않은 사용자 경험)

## 2. 모집 채널 (제안)

- 학교 교사 커뮤니티 (인디스쿨, 에듀콜라 등) 게시판
- 옵시디언 한국 사용자 모임 (디스코드·텔레그램)
- 트위터·블로그 — Sowing 소개 글 + 모집 링크
- 직접 추천 (개인 네트워크)

## 3. 모집 글 템플릿

```markdown

등을 자동 정리해 드립니다 (단, *사용자 명시 클릭이 있어야만* 정식 기록이
```

## 4. 선정 후 절차

1. **메일 회신** — 환영 + 설치 가이드 (`docs/SETUP.md` 또는 `docs/BETA_GUIDE.md`)
2. **30분 화상 미팅** (선택) — Sowing 작동 시연 + 첫 vault 셋업 도움
3. **카카오톡/디스코드 그룹** — 베타 5명 + 운영자, 질문·버그 즉시 응답
4. **2주 후 첫 체크인** — 설치·온보딩 완료 확인, 첫 합성 결과 본 소감
5. **한 학기 후 인터뷰** — `docs/BETA_GUIDE.md` §6 의 6 질문

## 5. 데이터 수집 (사용자 동의 후)

베타 사용자가 공유 동의한 경우:
- `bundle exec rake stats:synth_metrics` 결과 (수치만, 본문 X)
- `vault/.sowing/audit.log` 의 synth_* 라인 (개인정보 없음)
- 인터뷰 녹음·노트

**동의 없이 수집 안 함**: 본문 / 학생 이름 / 학교명 / 개인 정보.

## 6. 인터뷰 결과 → ROADMAP 마일스톤 갱신

베타 종료 후 운영자가 할 일:

```sh
# 1. 각 사용자의 데이터 집계
SOWING_SINCE=2026-03-01 SOWING_UNTIL=2026-07-31 \
  bundle exec rake stats:beta_report > beta-report-2026-1학기.md

# 2. 인터뷰 결과 정리 (별도 마크다운)

# 3. ROADMAP 마일스톤 블록 갱신:
#    - Phase 11 마일스톤 — 수락률·통찰 회고 결과 기록
#    - Phase 12 마일스톤 — "보고서 80% 작성" 회고 결과 기록

# 4. 부족한 부분이 발견되면 Phase 13 (혹은 확장 합성기) task 정의
```

## 7. ADR-013 약속 — 베타 운영 시 지켜야 할 것

베타 사용자에게 한 약속:
- ✅ **데이터 외부 전송 없음** (LLM 모드 옵트인 시만 본문 일부 LLM API 로 전송)
- ✅ **언제든 탈퇴 가능** — vault 폴더 그대로 유지 (옵시디언으로 계속 사용)
- ✅ **개인정보 보호** — 학생 실명·학교명 절대 수집 안 함
- ✅ **자율 mutation 0** — 사용자가 거절한 합성은 정식 기록이 되지 않음

운영자가 위반하면 ADR-013 자체가 깨짐 — 절대 어기지 말 것.
