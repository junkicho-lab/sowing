# Sowing 🌱 Phase 13 베타 인터뷰 가이드

> **대상**: v0.1.3 (2026-05-11) 출시 후 한 학기 (3~6개월) 사용한 베타 테스터 5명
> **목적**: Phase 13 (동사 중심 IA + Plan mode + 17번째 합성기) 의 정량·정성 검증
> **소요**: 30분 / 인 (사전 측정 + 라이브 인터뷰)
> **자동화**: `bin/sowing-doctor` + `/synth/metrics` + audit.log 로 정량 자동 추출

---

## 1. 측정 목표 (REDESIGN_IA.md §10)

Phase 13 의 정량 목표 — 인터뷰 끝나면 5명 평균이 임계값을 통과하는지:

| 지표 | 현재 (v0.1.1 기준 추정) | 목표 (v0.1.3 후) | 측정 방법 |
|---|---|---|---|
| 첫 메모까지 시간 | ~2분 | **< 30초** | onboarding 영상 분석 (사전) |
| 1주차 이탈률 | ~40% | **< 20%** | audit.log 의 마지막 활동일 |
| Nav 평균 hover 횟수 | 3.2회 | **< 1.5회** | 사용성 테스트 (라이브, 5명 평균) |
| 합성기 월 1회 이상 | ~30% | **> 60%** | `/synth/metrics` 의 wow_users |
| 메모/필기/기록 mode 의식 빈도 | 매번 | **거의 0** | UX 인터뷰 (정성) |

## 2. 사전 준비 (인터뷰 전 — 베타 테스터 셀프)

인터뷰 1주일 전 베타 테스터에게 이메일 안내:

```
안녕하세요,

Sowing v0.1.3 (Phase 13) 사용 의견을 나눠 주시기 전, 다음 한 가지만
준비 부탁드립니다 — 시간은 5분이면 충분합니다.

1. Sowing 폴더 위치 확인:
   ~/Documents/SowingVault/.sowing/audit.log

2. 한 명령어 실행 — 정량 데이터 추출 (저희에게 익명으로 보내주실 것):

   bin/sowing-doctor --beta-export > /tmp/sowing-beta-{본인이름}.json

3. 인터뷰 때 위 파일을 화면 공유해 주시면 됩니다.

본 데이터에는 **메모 본문 0건 포함되지 않습니다** — 카운트와 시간만.
audit.log 자체에도 entry_id (ULID) + path 만 있고 본문은 없습니다.

— Sowing 팀
```

> 💡 `--beta-export` 플래그는 W28-T05 (후속 작업) 에 추가 예정. PoC 단계에선 인터뷰 시점에 함께 실행.

## 3. 인터뷰 구성 (30분)

### Stage 1 — Warm-up (3분)

- "오늘 어떤 메뉴부터 보시나요?" — 자연스러운 사용 패턴
- "Sowing 안 쓴 날 있었나요? 며칠 정도?" — 이탈 신호 사전 감지

### Stage 2 — Nav 직관성 (5분, W25-T01·T02 검증)

```
이전 nav (10 항목 평면):
홈 · 메모 · 필기 · 기록 · 태그 · 검색 · 템플릿 · 합성기 · 그래프 · 설정

새 nav (5 동사 + 홈·설정):
🏠 홈 · 🖊 글쓰기 ▾ · 📚 쓴 글 보기 ▾ · 🗓 쓸 글 계획 ▾ · 🪞 자기 거울 ▾ · ⚙ 설정
```

**Q1**: "이전 vs 새 nav — 어느 쪽이 직관적이셨어요? 왜요?"
- 5점 척도 (1=훨씬 이전 / 3=비슷 / 5=훨씬 새 nav)
- 자유 답변 필수

**Q1-follow-up**: "변경 안내 모달 (한 번 본 뒤로 안 나오는) — 도움됐나요?"
- 1회 표시 + 닫기 패턴 평가

**Q1-실측**: 사용성 테스트 — "오늘 적은 메모 다시 찾아보세요"
- nav hover 횟수 측정 (관찰자 카운트)
- 목표: < 1.5회 평균

### Stage 3 — 글쓰기 진입 (5분, W26-T01·T02 검증)

**Q2**: "글쓰기 dropdown — 빠른 메모 외 책·강의·감정·학생 chip 써보셨나요?"
- 자주 쓴 chip 1순위 / 한 번도 안 쓴 chip
- "필요 없다 / 필요한데 잘 모르겠다 / 자주 쓴다" 3구간

**Q2-follow-up**: "감정 18종 chip — 너무 많나요? 빠진 거 있나요?"
- 의외로 자주 누른 감정 / 한 번도 안 쓴 감정

**Q3**: "음성 입력 🎙 — 시도해 보셨나요? 정확도는?"
- ko-KR Web Speech API 정확도 평가
- Chrome/Edge 이외 브라우저 사용 여부
- "음성이라 더 자주 적게 됐다" vs "텍스트가 편하다"

### Stage 4 — 쓸 글 계획 mode (5분, W27 검증)

**Q4**: "쓸 글 계획 mode — 실제로 쓰셨나요? 어느 period?"

| period | 사용 빈도 (체크) |
|---|---|
| 📅 일간 (오늘 할 일) | ☐ 매일 ☐ 주 2~3회 ☐ 거의 안 씀 |
| 📋 주간 | ☐ 매주 ☐ 가끔 ☐ 안 씀 |
| 🎯 월간 | ☐ 매월 ☐ 가끔 ☐ 안 씀 |
| 🏗 프로젝트 | ☐ 활용 중 ☐ 시도 ☐ 안 씀 |
| 🎓 학기 | ☐ 학기초 작성 ☐ 안 씀 |

**Q4-follow-up**: "대시보드 '오늘 할 일' 위젯 — 도움됐나요, 부담이었나요?"
- 매일 첫 화면 노출의 동기부여 vs 압박 측정

**Q4-실측**: 베타 테스터의 `40_Plans/` 폴더 확인
- 실제 작성된 plan 수 + period 별 분포
- 완료 토글 사용 빈도

### Stage 5 — 자기 거울 5축 (5분, W28 검증)

**Q5**: "🪞 자기 거울 — 매일 자동 생성 옵션 켜셨나요? 켜셨다면 의미 있었나요?"
- opt-in 비율 (5명 중 몇 명)
- 매일 5축 카드 노출의 효과:
  - "왜 적어야 하는지 동기가 생겼다" vs "또 봐야 하나 부담이다"

**Q5-follow-up**: "5축 (지성·감정·습관·관계·에너지) 중 가장 인상 깊었던?"
- 단일 선택 + 이유

**Q5-LLM**: "API 키 설정해서 LLM 5축 해석까지 보신 분 있나요?"
- 결정적 (무료) vs LLM (비용) 선호도
- "단정 거부 톤" 이 실제로 그렇게 느껴졌는지

### Stage 6 — 통합 의식 (4분)

**Q6**: "메모/필기/기록의 차이 — 의식하셨어요?"
- "거의 의식 안 했다 / 그래도 한다 / 매번 고민" 3구간
- Plan 까지 4 mode 됐는데 차이 인지하는지

**Q6-follow-up**: "옵시디언과 양방향 사용해 보셨나요?"
- 40_Plans/ 폴더가 옵시디언에서 어떻게 보였는지
- todo 체크박스 (`- [x]`) 동기화 확인

### Stage 7 — 정량 데이터 확인 (3분)

화면 공유 — 베타 테스터의 `/synth/metrics` 페이지 함께 보기:

- **수락률 %** — Phase 11 마일스톤 ≥50% 통과 여부
- **type 별 활용** — 17 합성기 중 사용한 type 수
- **자기 거울 사용** — 매일 자동 + 수동 합산

audit.log 직접 확인 (옵션):

```sh
# 1주차 이탈률 baseline
last_activity=$(jq -r '.at' ~/Documents/SowingVault/.sowing/audit.log | tail -1)
# 최근 활동일 vs 오늘 — 7일 이상 차이면 이탈 신호
```

## 5. 의사결정 트리

인터뷰 결과 → 다음 단계:

| 결과 | 다음 단계 |
|---|---|
| 모든 정량 ≥ 목표 + 정성 긍정 | Phase 14 진입 (베타 → 정식 출시 준비) |
| 정량 ≥ 목표 + 일부 정성 부정 | 부정적 영역 patch (예: 자기거울 부담 → opt-in 강조) |
| 정량 미달 + 명확한 원인 | Phase 13 후속 (예: nav 의 특정 dropdown 개선) |
| 정량 미달 + 원인 불명 | 베타 확대 (10명) 후 재인터뷰 |

## 9. 부록 — 인터뷰 진행자용 cheat sheet

### 좋은 답변을 끌어내는 방법

- ❌ "Nav 좋으셨어요?" (yes/no)
- ✅ "어떤 nav 메뉴를 가장 자주 클릭하셨어요? 왜요?" (구체)

- ❌ "자기 거울 도움됐죠?" (유도)
- ✅ "자기 거울 결과 보고 '아 이거 좀 신경 써야겠다' 라고 생각하신 적 있나요?" (행동 변화)

### 침묵 활용

- 질문 후 7초 침묵 — 답 짧으면 더 깊은 답 끌어옴
- "더 있으세요?" / "예시 하나 더?"

### 데이터 공유 동의

- audit.log + entries 통계는 익명화 후 집계만 — 개인 본문 0 노출
- 베타 테스터 1인 응답이 다른 4명에게 공개 안 됨 (각자 별도 인터뷰)
- 보고서 작성 시 인용 동의 별도 확인
