# Sowing 베타 사용자 가이드

> 한 학기 (3~6개월) 사용 후 합성기 검증을 도와주실 베타 테스터를 위한 안내.

## 1. 베타 테스터에게 부탁드리는 것

1. **평소처럼 메모·필기·기록을 작성** — 학교에서 일상적으로 쓰는 그대로. 의식적으로 "AI 친화적" 으로 쓰실 필요 없습니다.
2. **합성 결과를 검토하고 수락/거절** — `/synth` 라우트의 12 type 합성기 산출물을 보고 정직하게 판단:
   - **수락**: "이 합성이 실제로 도움 됐다" → `30_Records/{YYYY}/{category}/` 으로 보존
   - **거절**: "쓸모 없다 / 부정확하다 / 오해 소지 있다" → `.sowing/trash` 휴지통
3. **한 학기 후 짧은 인터뷰** (15~30분) — 어떤 합성기가 가장 유용했는지, 어떤 것은 무엇이 부족했는지 의견 공유.

## 2. Phase 11/12 마일스톤 측정 기준

| Phase | 지표 | 임계값 |
|-------|------|--------|
| 11 | 학생 디제스트 사용자 수락률 | ≥ 50% |
| 11 | "내가 손으로 못 합쳤을 통찰을 얻었다" 회고 | ≥ 3건 |
| 12 | "학기말 회고 보고서 80% 작성됐다" 회고 | ≥ 3건 |

수락률은 `audit.log` 의 `synth_accept` / `synth_reject` 비율로 자동 측정.
회고는 인터뷰로 수집.

## 3. 측정 도구

### `/synth/metrics` — 실시간 대시보드
브라우저에서 `/synth/metrics` 진입:
- 전체 수락률 + Phase 11 마일스톤 ✅/🟡 자동 표시
- type 별 활용 분포 (학생/학기/패턴/변화/상담/평가/연수/주간/고립/시리즈/클러스터/계절성)
- 주별 추이 (최근 8주)

### `bundle exec rake stats:synth_metrics` — CLI 리포트
터미널에서 같은 데이터를 텍스트로:
```sh
bundle exec rake stats:synth_metrics
```

기간 지정:
```sh
SOWING_SINCE=2026-03-01 SOWING_UNTIL=2026-07-31 bundle exec rake stats:synth_metrics
```

### `bundle exec rake stats:beta_report` — 마크다운 리포트
인터뷰 자료·운영 보고용 마크다운 파일:
```sh
bundle exec rake stats:beta_report > beta-report-2026-1학기.md
```

## 4. ADR-013 거부 5종 — 베타 기간에도 동일 보장

베타 사용자에게 약속드리는 5가지:

1. ❌ **챗봇 UI 없음** — Sowing 안에서 LLM 과 대화하지 않음. 합성 결과만 받음.
2. ❌ **자동 글쓰기 거부** — 일반 메모/필기/기록을 LLM 이 대신 쓰지 않음. 합성기 산출물은 별도 `.sowing/synth/` 격리.
3. ❌ **클라우드 LLM 강제 안 함** — 모든 LLM 기능은 옵트인. 결정적 모드(LLM 0)도 모든 합성기에서 동등 지원. Ollama 등 로컬 LLM 동등 지원.
4. ❌ **의인화 카피 없음** — "AI가 ~ 생각합니다" 같은 표현 없음. "LLM 합성" 배지 명시.
5. ❌ **자율 mutation 0** — 사용자 명시 클릭 없이는 vault 변경 안 함. 합성 → 검토 → 수락/거절 항상 사용자 결정.

## 5. 사적 데이터 보호

- 모든 데이터는 **본인 컴퓨터의 vault 폴더에만 저장** — 외부 서버 전송 없음.
- LLM 모드 사용 시 본문 일부가 LLM API 로 전송되지만, 사용자가 명시적으로 LLM backend 를 활성화한 경우만.
- audit.log 는 **로컬 파일** — 베타 보고를 위해서는 사용자가 직접 일부 수치만 공유 (`rake stats:synth_metrics` 결과의 type별 카운트).
- 학생 실명 노출이 우려되면 vault 의 학생 이름을 가상명·별칭으로 작성 권장 (`Settings → 학급 명단` 도 가상명으로).

## 6. 인터뷰 질문 (한 학기 후)

운영자가 베타 사용자에게 물어볼 핵심 질문 6개:

1. 12 합성기 중 가장 유용했던 3개는? 왜?
2. 가장 쓸모없었던 (또는 거절한) 3개는? 왜?
3. "내가 손으로 못 합쳤을 통찰" 을 받은 사례가 있었나요? 구체적으로 어떤 것?
4. 학기말 회고 합성을 학교 보고서에 사용했다면, 몇 % 정도 활용됐나요?
5. ADR-013 의 "자율 판단 0" 원칙이 실제로 지켜졌다고 느꼈나요?
6. 다음 학기에 어떤 합성기를 추가하면 좋을까요?
